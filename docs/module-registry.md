Module Registry Overview

The complete, auto-generated registry that Claude Code and the validator use lives at:
- src/lib/workflows/module-registry.ts
- It is generated by scripts/generate-module-registry.ts (run: npm run generate:registry)

Module path format
- category.module.function
- Example: utilities.math.add → src/modules/utilities/math.ts → export function add(a: number, b: number): number

Function input/output shapes
- Signatures are included in module-registry.ts as strings (e.g., signature: "generateText(options)").
- Many modules accept object wrappers: params or options. In workflows, wrap arguments accordingly: inputs: { options: { ... } } or inputs: { params: { ... } }.
- For functions with multiple primitive parameters (e.g., add(a, b)), pass named inputs matching parameter names. The executor will map to signature order automatically.

How to regenerate the registry
- npm run generate:registry
- This scans src/modules and writes src/lib/workflows/module-registry.ts

Notes
- The executor has auto-injection for API key variants of read-only functions (e.g., get/list/search) when a WithApiKey version exists.
- AI SDK steps: executor can inject apiKey based on model/provider from loaded credentials if options.apiKey is omitted.
